iManage Documentation

Welcome to the official technical documentation for iManage, a desktop CRM application for car rental management. This document serves as both a technical reference and an educational guide for developers.

1. System Architecture

iManage is a hybrid desktop application built with web technologies. It runs locally with no cloud dependencies.

Container Electron Packages the app as a native desktop executable.

Backend Express.js REST API server; handles business logic.

6. Developer Journey

This section is for aspiring developers who want to understand how to think like an engineer, not just copy-paste code.

The Restaurant Analogy

Concept: Separation of Concerns

Customer (User) looks at the Menu (Frontend).

When they order (e.g., "Create Invoice"), they call the Waiter (API Request).

The Waiter goes to the Kitchen (Backend).

The Chef uses ingredients from the Pantry (Database).

The Waiter brings back the finished dish (API Response).

Vibe Coder puts database code inside React components. üî•

Developer sends a request to the backend. The backend handles the risky stuff. ‚úÖ

Level Up: Ask "What If?"

  • What if the database file is locked?
  • What if the user enters an emoji as a phone number?
  • What if the network fails mid-request?

Developers anticipate failure. Vibe coders pray it works.
