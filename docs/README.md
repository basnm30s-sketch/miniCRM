## Documentation Standards

### File Naming
- Use UPPERCASE for main documentation files
- Use lowercase with hyphens for specific guides
- Include dates in filenames for time-sensitive docs

### Content Structure
- Start with overview/executive summary
- Include detailed sections
- Add diagrams where helpful
- Include code examples
- Document known issues and limitations

### Maintenance
- Update documentation when features change
- Keep changelog current
- Update architecture docs for major changes
- Review and update quarterly

## Contributing to Documentation

When adding new features:
1. Update [CURRENT_STATE.md](./CURRENT_STATE.md)
2. Update [CHANGELOG.md](./CHANGELOG.md)
3. Update [ARCHITECTURE.md](./ARCHITECTURE.md) if architecture changes
4. Add test documentation if applicable
5. Update relevant module documentation
